## 2. CHI TIẾT CÁC GIAI ĐOẠN THỰC HIỆN (STAGES)

### Giai đoạn 1: Phân tích yêu cầu
- Yêu cầu chức năng và phi chức năng: Đặc tả chi tiết các nghiệp vụ hệ thống quản lý đăng ký học phần.
- Sơ đồ phân tích: Thực hiện vẽ DFD, Use Case, Class Diagram trên công cụ app.diagrams.net (Draw.io).

### Giai đoạn 2: Thiết kế hệ thống
- Cơ sở dữ liệu: Thiết kế Data Model mức Logic. Lưu trữ dữ liệu thông qua các tập tin văn bản (.txt) theo đúng yêu cầu.
- Giao diện (UI): Mô tả luồng Menu điều hướng người dùng bằng văn bản tiếng Việt chi tiết cho môi trường Console.

### Giai đoạn 3 & 4: Cài đặt và Kiểm thử
- Dữ liệu mẫu: Chuẩn bị sẵn dữ liệu trong các file .txt để demo trực tiếp, đảm bảo không nhập tay từ đầu khi báo cáo.
- Mã nguồn: Triển khai trên ngôn ngữ C++ theo phương pháp lập trình hướng đối tượng (OOP).
- Kiểm thử: Chạy các kịch bản Test Case để đảm bảo logic đăng ký môn học và xử lý sĩ số chính xác.

## 3. HƯỚNG DẪN CÀI ĐẶT VÀ VẬN HÀNH
Để Giảng viên chấm bài trực tiếp, vui lòng thực hiện các bước sau:

1. Sao chép mã nguồn (Clone): git clone https://github.com/levanthanh72/Project-Course-Registration-System.git
2. Cài đặt: Mở Project bằng Visual Studio, kiểm tra sự tồn tại của các file dữ liệu .txt trong thư mục dự án.
3. Khởi chạy: Nhấn F5 để biên dịch và chạy chương trình trên môi trường Console.
